状态说明：

## 策略一：资源归一化降维

**状态：[已实现] — Normalizer 模块**

### 思路
将 N 个资源约束合并成 1 个归一化资源，把 N 维 RCSPP 转化为 1 维 RCSPP。

### 方法
1. 采样所有边的资源消耗，取第 20/40/60/80 百分位的平均值作为归一化系数 coeff[r]
2. 构建归一化图，每条边的归一化资源 = sum( weight[r] / coeff[r] × original[r] )
3. 归一化资源限制 = sum( weight[r] / coeff[r] × limit[r] )
4. 在归一化图（2维：主代价 + 1个归一化资源）上运行 ERCA*

### 效果
- 归一化图上的 ERCA* 找到的路径若在原图可行，则该路径直接作为精确上界
- 实验验证：找到的路径 cost ratio 始终 = 1.000（即恰好是最优路径）

### 局限
- 归一化系数基于边的统计分布，不一定准确反映路径层面的权衡
- 资源之间负相关时归一化效果变差（组内资源此消彼长）

## 策略二：迭代权重调整

**状态：[已实现] — Normalizer 模块**

### 思路
归一化得到的路径可能在原图中违反某些资源约束，通过多次迭代调整各资源的权重，直到找到可行路径。

### 方法
```
初始 weight[r] = 1.0（所有资源等权）

循环最多 maxIter 次（默认5次）：
  构建归一化图并求解
  验证路径在原图是否可行：
    若可行 → 返回上界，结束
    若资源 r 超限（ratio > 1.0） → weight[r] *= 2.0
    若资源 r 宽裕（ratio < 0.5） → weight[r] *= 0.5
```

### 效果
- 紧约束（alpha < 0.4）时，第1次迭代成功率约 11-20%，多次迭代后可提升
- 宽松约束（alpha >= 0.3）时，100%成功率，迭代1次即可

### 局限
- 每次迭代都要重建归一化图并运行一次 ERCA*，代价不低
- 权重调整是启发式的，不保证一定找到可行解

## 策略三：Pulse DFS 求上界

**状态：[已弃用] — Pulse 模块**

### 思路
在归一化图上用深度优先搜索（DFS）快速找到一条可行路径，作为代价上界。参考 Lozano & Medaglia (2013) 的 Pulse 算法。

### 为什么弃用
Pulse DFS 在网格图上搜索空间指数爆炸：

```
网格图节点多、分支因子大，DFS 无法有效剪枝
测试中所有案例均超时（设置5秒超时限制）
```

### 替代方案
用 ERCA* 在归一化图上求解（策略一），更可靠。

## 策略四：上界剪枝（代价上界 UB）

**状态：[已实现] — ERCA 模块**

### 思路
在 ERCA* 搜索过程中，若已知一条可行路径的代价为 UB，则所有 f 值超过 UB 的标签必然不优，可以直接剪掉。

### 方法
```
在 ERCA* 的 resourceCheck 函数中增加：
  if (label.f[0] > upperBound[0]) → 剪枝（计入 nPrunedByUpperBound）
```

上界 UB 来自归一化 ERCA* 的解（策略一）。

### 关键理论发现
**一致启发式下，UB 无法减少已展开节点（只能减少已生成标签）。**

原因：ERCA* 按 f 值升序展开，若启发式一致（h ≤ 真实代价），则展开顺序保证最先到达目标的就是最优路径，UB ≥ 最优代价，不会剪掉任何已展开节点。

**因此 UB 剪枝必须配合启发式膨胀才有效。**

## 策略五：启发式膨胀（Weighted A*）

**状态：[已实现] — ERCA 模块**

### 思路
将启发式函数乘以膨胀系数 w > 1：

```
原始：f(label) = g(label) + h(label)
膨胀：f(label) = g(label) + w × h(label)，w = 1.5
```

膨胀后启发式不再一致，ERCA* 变为 Weighted A*，可以更激进地剪枝。

### 效果（消融实验结果）
```
w = 1.0（不膨胀）：节点缩减 0%（UB 无效）
w = 1.5（膨胀）：节点缩减 83-92%（大量剪枝）
w = 1.5 + UB：节点缩减 100%（宽松约束下完全不展开）
```

### 代价
- 解不再保证最优（近似解）
- 实验中 cost ratio 约为 1.00-1.11（最差约11%偏差）

## 策略六：智能阈值（alpha 难度系数）

**状态：[已实现] — smart_benchmark 实验程序**

### 思路
用资源限制的"语义化"参数 alpha 替代人工设定的松紧度 t，使不同图上的难度具有可比性。

### 方法
```
LB[r] = 反向 Dijkstra（按资源维度 r）→ 从 target 到 source 的最小资源消耗
UB[r] = 正向 Dijkstra（按主代价维度 0）→ 代价最优路径在资源 r 上的消耗

Limit[r] = LB[r] + alpha × (UB[r] - LB[r])
```

### alpha 的含义
```
alpha = 0.0 → Limit = LB（最严苛，几乎无可行解）
alpha = 0.5 → 中间点（通常是"甜点"测试区）
alpha = 1.0 → Limit = UB（代价最优路径刚好可行）
```

### 实验发现
- UB/LB 比值约 1.6-2.1，对应旧参数 t ≈ 1.6-2.1
- alpha = 0.4 是搜索难度最大的甜点（84%可解率，节点数最多）
- alpha = 0.4-0.6 时，纯膨胀策略实现 4-5x 时间加速

## 策略七：部分降维（N 维 → K 维归一化）

**状态：[理论] — 未实现**

### 思路
将现有的"N→1 完全压缩"推广为"N→K 部分压缩"：把 N 个资源分成 K 组，每组内归一化，最终在 K 维图上运行 ERCA*。

### 复杂度分析
```
ERCA* Pareto 前沿大小：O(W^(K-1))

K = 1（当前方法）：O(1)，最快，但信息损失最大
K = N（精确 ERCA*）：O(W^(N-1))，最慢，无信息损失
K = 中间值：权衡点
```

N=8 时，K 从 8 降到 3，复杂度从 W⁷ 降到 W²，相差 W⁵ 倍。

### 分组策略（数学推导结论）

**分组依据：资源之间的相关系数**

```
相关系数 ≈ +1（正相关）→ 可合并，信息损失小
相关系数 ≈  0（独立）  → 应分开，合并会丢信息
相关系数 ≈ -1（负相关）→ 必须分开，合并破坏 Pareto 结构
```

**对完全独立资源的严格证明：**

设 K 组的大小分别为 n_1, n_2, ..., n_K，sum = N。

由 AM-GM 不等式：
- Product(n_g) 在各组等大（n_g = N/K）时取最大值
- Product(n_g) 在最不均等时取最小值

而 Pareto 前沿大小 ∝ Product(n_g)，因此：
- 计算效率：不均等分组更优
- 上界准确率：均等分组更优（等大分组误差最小）
- 综合最优：由对称性论证，独立资源应均等分组

### 最优 K 的选择

**方法一：PCA 解释方差**
对所有边的资源权重矩阵做 PCA，选最小的 K 使累计解释方差 ≥ 90%。

**方法二：计算预算**
给定时间上限 T_max，解：K ≤ log(T_max/n) / log(W) + 1

**方法三：有效秩**
特征值分布的指数熵，给出资源的"有效独立维数"。

### 与现有方法的关系
```
现有方法（K=1）：最快，信息损失最大
部分降维（K=自适应）：自动平衡精度与速度
精确 ERCA*（K=N）：最慢，无信息损失
```

## 关键实验结论

**1. 什么情况下归一化有正收益？**
- 图规模 > 1000 节点
- 约束偏紧（alpha ≈ 0.4-0.5，对应 t ≈ 1.5x）
- 35x35 网格 / alpha=0.5 实现了 **1.37x** 加速

**2. 什么情况下直接用膨胀更好？**
- 图规模 < 1000 节点（归一化开销 > 收益）
- 任何规模的图，追求零预处理延迟时

**3. 归一化迭代次数的贡献**
- 宽松约束（alpha ≥ 0.3）：第1次迭代100%成功，后续迭代无意义
- 紧约束（alpha ≈ 0.4-0.5）：前2次迭代贡献了大部分加速
- 建议：宽松时固定1次迭代，紧约束时最多3次

**4. 独立资源与相关资源的根本区别**
- 资源正相关 → 归一化有效，上界准确
- 资源独立 → 归一化信息损失大，K 维部分降维是更好选择
- 资源负相关 → 归一化可能破坏 Pareto 结构，必须分开处理
